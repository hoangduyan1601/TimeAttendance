BÁO CÁO TIẾN ĐỘ XÂY DỰNG BACKEND - TUẦN 4
Dự án: Hệ thống quản lý chấm công thông minh SmartOps

I. TỔNG QUAN KIẾN TRÚC
- Kiến trúc: Microservices độc lập.
- Core Backend: Java Spring Boot (v3.4.0), JDK 17.
- AI Microservice: Python FastAPI (v3.12), thư viện DeepFace (Facenet model).
- Cơ sở dữ liệu: PostgreSQL (Quản lý User, FaceData, Shift, Attendance Logs).
- Lưu trữ ảnh: Local Storage (Tối ưu chi phí, đã cấu hình Static Resource mapping để truy cập qua URL).

II. DANH SÁCH API ĐÃ HOÀN THIỆN (100% THEO THIẾT KẾ)

1. Phân hệ Core Backend (Cổng 8081)
- [POST] /api/v1/auth/login: Xác thực người dùng và cấp phát JWT Token.
- [POST] /api/v1/auth/ekyc: Nhận ảnh CCCD & Selfie, gọi AI để mã hóa khuôn mặt và lưu vào DB.
- [GET]  /api/v1/auth/qr-code: Sinh mã QR động (User ID + Timestamp) có hiệu lực 30 giây.
- [POST] /api/v1/kiosk/verify: Xác thực chấm công (Giải mã QR + So sánh khuôn mặt AI + Kiểm tra ca làm việc).
- [GET]  /api/v1/kiosk/live-logs: Lấy 10 bản ghi chấm công mới nhất theo thời gian thực.
- [GET]  /api/v1/admin/dashboard/stats: Thống kê Sĩ số, Muộn, Vắng, Đơn từ trong ngày.
- [POST] /api/v1/admin/attendance/adjust: Hiệu chỉnh giờ công thủ công (Admin).
- [GET]  /api/v1/admin/reports/export: Kết xuất báo cáo chấm công ra file Excel (.xlsx).
- [CRUD] /api/v1/admin/users: Quản lý thông tin nhân sự.
- [CRUD] /api/v1/admin/shifts: Cấu hình khung giờ ca làm việc.

2. Phân hệ AI Microservice (Cổng 8000)
- [POST] /internal/ai/embed: Trích xuất Vector khuôn mặt 128 chiều từ ảnh.
- [POST] /internal/ai/compare: So sánh Vector gốc với ảnh trực tiếp, trả về độ tương đồng (Similarity).

III. CÁC ĐIỂM NHẤN KỸ THUẬT ĐÃ ĐẠT ĐƯỢC
1. Sinh trắc học: Tích hợp thành công mô hình Facenet 128D, đảm bảo độ chính xác cao và tốc độ xử lý nhanh.
2. Bảo mật: Triển khai JWT Filter kết hợp Spring Security, bảo vệ toàn bộ API nghiệp vụ.
3. Linh hoạt: Cấu hình Static Resource Mapping cho phép xem ảnh trực tiếp qua URL, hỗ trợ tốt cho Flutter Web mà không tốn phí Cloud Storage.
4. Kiểm thử: Đã xây dựng script tự động (Python) kiểm tra thành công toàn bộ luồng từ Đăng nhập -> eKYC -> Chấm công Kiosk.

IV. KẾT QUẢ KIỂM THỬ THỰC TẾ
- Đăng nhập/Cấp Token: Thành công.
- Trích xuất Face Vector: Thành công (128 chiều).
- So sánh khuôn mặt tại Kiosk: Thành công (Độ tương đồng 1.0 cho ảnh gốc).
- Ghi log chấm công: Thành công (Tự động xác định trạng thái ĐI MUỘN/ĐÚNG GIỜ dựa trên Shift Config).
- Thống kê Dashboard: Dữ liệu cập nhật realtime sau mỗi lần chấm công thành công.

V. KẾ HOẠCH TIẾP THEO
- Xây dựng giao diện Flutter App (Nhân viên) và Flutter Web (Admin/Kiosk).
- Tích hợp gọi API từ Frontend lên Backend.
